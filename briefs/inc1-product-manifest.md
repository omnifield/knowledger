## Что заказываем

Спроектировать **КОНТРАКТ тонкого манифеста продукта** — визитку, из которой хаб и оркестрация узнают ровно **необходимое, не больше**.

## Канон (из блюпринта — НЕ нарушать)

- **Продукт самодостаточен**: манифест ТОНКИЙ; расширенные настройки живут ВНУТРИ продукта. Продукт поднимается сам `docker compose up` без хаба.
- **Хаб опционален**: манифест = интерфейс интеграции продукт↔хаб, НЕ хранилище настроек продукта.
- **devopser агрегирует**, не генерит внутренности продукта. Хаб/оркестрация только ЧИТАЮТ манифест.

## Поля (черновой набросок из блюпринта — уточнить/финализировать)

- **identity:** `name` · `type` (frontend/backend/fullstack/service)
- **reach:** `gatewayRoute` · `port(s)`
- **integration:** `scopes[]` · `spawnEligible` · `deps[]` (на другие продукты, если есть)
- **(возможно)** `title`/`description` для карты хаба.

Обязательные vs опциональные — определить. Держать МИНИМАЛЬНЫМ (соблазн «дописать ещё поле» = утечка расширенного внутрь манифеста, стоп).

## Вопросы к решению (knowledger)

1. **Формат/имя файла** — блюпринт: отдельный чистый файл (НЕ nx `project.json`). `omnifield.yaml`? расширение, место (корень репо продукта).
2. **Не-JS продукты** (Go-chater, Python-backend) несут тот же файл в корне — подтвердить универсальность.
3. **Кто что читает** — разграничить: **хаб (тонкий слой воркспейса)** читает identity+reach+integration для реестра/карты; **спавн агентов — капабилити ПРОДУКТА brainer** (втянут в хаб), не функция хаба; **devopser-продукт** читает reach для gateway/compose-агрегации. (Хаб ≠ brainer ≠ devopser — поправка модели 2026-07-12.) Чтобы «тонкость» не поехала.
4. **Версионирование контракта** — публикуется как `@omnifield/contract-*` или наследуется через standards? (README knowledger: contracts = версионируемые мосты.)
5. **Граница с dev-сервисами** — как продукт объявляет autostart-сервисы: в манифесте (тонко: имя+порт) ИЛИ отдельно внутри продукта (`devbox.services.json` из devopser-дизайна `devbox-first-run-dx.md`)? **Согласовать с devopser, без дублирования.**
