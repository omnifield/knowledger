# POLICY — верховные правила Omnifield

Наивысший приоритет. Любой агент (architect любого репо, owner, layer-агент), начинающий работу, обязан соблюдать. Детали канона — в `canon/`, дисциплины — в `workflow/`.

---

## 0. 🧱 КАНОН — модули, не монолит

Экосистема из **независимых** модулей, а не монолит-in-disguise. Каждый пакет/сервис самодостаточен; один не пускает корни в другой. Совместимость — свойство **дизайна**, не патчей сверху (без адаптеров, шимов, костылей).

- Контракт между зонами — через `commons/contracts` (typed, версионируемый), НЕ через прямой импорт реализации соседа.
- `backend/` ⊥ `framework/` ⊥ `apps/` — связь только через опубликованные пакеты и контракты.

Детали: `canon/principles/modules-no-crutches.md`.

## 1. 🛡️ НИКАКИХ КОСТЫЛЕЙ

Работает не так как должно → **чиним причину до конца, тестим, идём дальше**. Не копим «временно оставлю». Если решение требует кучи нестабильных шагов / hardcoded путей / silent-fallback'ов / try-catch-swallow — **подход в корне неверный, СТОП, пересматриваем**. Omnifield — фундамент реальных продуктов; костыль во фреймворке = compound debt на всё, что им пользуется.

Детали: `canon/principles/root-cause-not-symptom.md`.

## 2. 🎯 ЭТАЛОН-ГЕЙТ («готово»)

«Готово» = **код + тесты + трейсы + доки + эталонная раскладка**. До этого — временное состояние, НЕ канон. Пакет доводится до эталона **на въезде** в v2 (не «потом причешем»).

Детали: `canon/principles/etalon-gate.md`, `canon/packages/anatomy.md`.

## 3. 🔄 МИГРАЦИЯ, НЕ REWRITE (v2)

Старый `capsule` = **оракул** (черновик): вычитываем при переносе, новые фичи туда не пишем. Логику по памяти НЕ переписываем — это ловушка rewrite'а (×2–3 времени, возврат багов, потеря momentum). Цикл каждого куска: **копия → верификация против оракула → эталон → доки → трейсы**.

Детали: ADR 077.

## 4. 🎩 OWNERSHIP — репо, зоны, роли

- **Architect репо** — триаж, арх-решения (cross-package/контракты/ADR), координация, следит за OWNERSHIP-границей. НЕ правит `src/` пакетов сам без причины — делегирует owner'у.
- **Owner зоны** — код внутри своего пакета/сервиса; читает OWNERSHIP.md + тесты green до изменений; обновляет тесты+доки на breaking change.
- **Эскалация снизу вверх**: owner → architect репо → (cross-repo) через контракт → product owner (user). Никогда наоборот.
- Cross-repo концерн → **контракт в `commons/contracts`**, не переброс брифами.

Детали: `canon/packages/ownership.md`, `agents/`.

## 5. 🚦 ДИСЦИПЛИНА ПЕРВОЙ

Настоящий фикс v1-проблем — процесс, не папки:
- **DoR/DoD по стадиям** — фичу вниз по потоку не начинают, пока не готов апстрим (флоу/БД/валидатор/фронт). Живой роадмап-статус.
- **Гигиена доков** — один источник правды на тему, явный `superseded`, без дата-археологии.
- **Протокол интеграций** — стандарт для внешних скоупов: онбординг + версионируемый контракт + канал.

Детали: `workflow/`, `integration-protocol.md`.

## 6. 🧪 ДВЕ ФОРМЫ ПРОВЕРКИ

| Форма | Где | Что проверяет |
|---|---|---|
| **Mechanics-testbed** | `framework/testbed/` — безликие боевые сценарии | механику **в сборе** (групповая работа компонентов, cross-component события) — то, что юнит-тесты изолированно не ловят |
| **Prod-условия** | `apps/` — потребляют published `@omnifield/*` | фича готова только когда работает у внешнего потребителя (публикационный дрейф пойман) |

Юнит-тесты пакета — необходимое, но НЕ достаточное. Оба яруса обязательны до «готово».

---

**При соблазне обойти правило** — не обходи. Правило мешает решить задачу = сигнал, что задача/канон требует пересмотра → **обсуждаем (ADR)**, не пишем второй workaround.
