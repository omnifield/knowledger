# Регистрация компонентов — 2 слоя

Компонент, который должен быть виден рендереру и стору studio, регистрируется в **двух слоях**. Один слой без другого = компонент мимо системы.

## Два слоя

1. **Manifest** — спецификация компонента (пропсы, слоты, пресеты, compound-состав) для studio: инспектор пропсов, палитра, генераторы. Декларативное описание «что можно настроить».
2. **`Ui`-namespace** — рантайм-регистрация в неймспейсе `Ui.*` (imports + interfaces + invariant-тест), чтобы проксированный `Ui` в слоях видел компонент и включал event-binding/meta-flow.

## Правило

Новый composite/primitive kit'а → **оба слоя** в одном заходе:
- manifest-запись (studio его увидит),
- `Ui`-namespace (рендерер/слои его отрисуют),
- invariant-тест зелёный (реестры не разъехались).

Только manifest → studio знает, но рендерер не рисует. Только namespace → рисует, но studio не инспектит/не кладёт в палитру. Инвариант-тест ловит рассинхрон.

## Почему

Studio-рендерер строит UI по JSON-схеме + Registry (компоненты по dot-path `ui.Button`). Manifest даёт studio редактируемую модель, namespace — исполняемую. Два слоя = компонент одинаково доступен и в коде (проксированный `Ui`), и в визуальном редакторе.
