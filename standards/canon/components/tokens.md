# Токены — frozen set, ноль raw-style

## Тезис

Визуал держится на **фиксированном наборе токенов** (цвета/spacing/typography/border/radius). Набор **frozen** — решаем задачу существующими токенами, не плодим новые под каждый кейс.

## Правила

- **Ноль raw-style.** Никаких `<style>`-блоков, inline-стилей, магических значений (`text-[10px]`, `bg-#abc`). Всё — через токены/пресеты kit'а.
- **Состояние в JS + токены.** Динамика (loading/active/disabled) — реактивный класс из токен-набора, инжектится kit-компонентом; не ручной стиль у потребителя.
- **Токен-сет заморожен.** Новый токен = осознанное расширение набора (owner-style + дизайн-синк), не ad-hoc значение в компоненте. Мешает существующий набор → обсуждаем расширение, не хардкодим.
- **Единый источник** — border/color/spacing токены определены один раз (напр. единый `--border`), не дублируются per-компонент.

## Почему

Raw-style и магические значения = визуальный шум, который нельзя рескинить и нельзя свести в дизайн-систему. Frozen token-set = гарантия, что весь продукт говорит одним визуальным языком; расширение — контролируемое, через дизайн-синк, а не «here подкручу пиксель». Часть kit-first ([[kit-first]]): классы в kit, значения из токенов.
