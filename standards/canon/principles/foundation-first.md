# Foundation-first — дыры закрываются ДО старта, не «по ходу»

Верхний закон — `../../POLICY.md`. Родственные: [root-cause-not-symptom](root-cause-not-symptom.md)
(лечим причину), [etalon-gate](etalon-gate.md) (что значит «готово»). Операционный гейт —
`../../workflow/definition-of-ready.md` (DoR).

## Тезис

**Не начинаем разработку, пока в базе есть известные дыры — даже если они не блокеры.**
Сначала база без дыр и костылей, потом фичи. «Не блокирует прямо сейчас» — не аргумент:
дыра, накрытая фичами, дорожает с каждым слоем сверху (compound debt) и всплывает в худший
момент — на другой машине, в CI, у второго продукта.

## Отличие от соседних гейтов

- **DoR «Upstream Done» / «нет блокирующих unknowns»** — гейтят *блокеры* конкретного item.
- **Foundation-first** — гейтит *известные дыры базы*, включая не-блокеры: «можем начать и
  так, дыра не мешает» → всё равно НЕ начинаем, дыру закрываем первой.

## Иерархия базы — что чью работу гейтит

```
commons (правила) → devopser (машины/инфра) → скелет репо → продуктовая разработка (фичи)
```

Дыра на уровне N блокирует старт **всего, что выше N, во всех репо**: дыра в devopser =
стоит вся продуктовая работа экосистемы, потому что devops — основа. Параллельность
допустима только **внутри уровня** (два стека devopser — можно; дыра в devopser + старт
скелета продукта — НЕЛЬЗЯ).

Дыра ≠ незавершённая запланированная работа: второе гейтит только своих потомков по
зависимостям (DoR «Upstream Done»), не весь уровень. Гейтит-всё именно **известное
несоответствие** (дыра).

## Пока база не зелёная — строгая сериализация (WIP = 1)

Пока в экосистеме есть открытые 🕳: **один активный item на всю экосистему**, снизу вверх
по иерархии. Дыро-фиксы — НЕ исключение и не повод для параллели: они тоже стоят на базе
(хуки, тулчейн, дерево) и проваливаются в её дыры точно так же. Параллельность — привилегия
зелёной базы: разрешается architect'ом явной записью в ROADMAP, только внутри уровня и
только с изоляцией working tree (shared-policy §4).

## How to apply

1. Нашёл дыру в базе (скелет репо, тулчейн, инфра, контракт) во время подготовки → фикс /
   бриф / эскалация — **до** старта зависимой работы. Item в roadmap = `blocked`, не `doing`.
2. Дыра в чужой зоне → эскалация вверх с матрицей решений (что/где/кто решает), работа стоит
   до отмашки. Это не «потеря темпа» — это и есть темп: полчаса на дыру сейчас vs дни на
   раскопки под фичами потом.
3. «Дыра или улучшение?» — тест: если это *известное* несоответствие канону/контракту или
   недекларированное поведение (версия нигде не запинена, warning вместо гейта, deprecated
   опора) — дыра. Если «можно сделать ещё лучше» без несоответствия — улучшение, в roadmap,
   не гейтит.
