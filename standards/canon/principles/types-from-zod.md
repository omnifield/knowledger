# Типы — только из zod, никогда руками

## Тезис

**Zod-схема — единственный источник типа.** Ручной `interface`/`type` для домена запрещён **везде**: app (View/Shape/Widget/Controller/Feature), пакет (Entity-потребители), контракт (`commons/contracts`). Нет ни одного легитимного кейса, где нужно писать `interface`/`type` руками под домен — это всегда `z.infer<typeof schema>`.

## Что разрешено

- **`z.infer<typeof schema>`** — тип, выведенный из zod-Entity/zod-контракта. Единственный источник домена.
- **Глобальные декларации** — ambient `.d.ts`, сгенерированные codegen-типы (напр. `OmnifieldSlots`).
- **Vendor-типы** — импортированные типы сторонних библиотек (не наш домен).

## Что запрещено

- Ручной `interface X {}` / `type X = {...}` под доменную сущность, props, контракт — на **любом** уровне (app/package/contract). Без исключений «просто маленький тип для пропа» — проп берётся из zod-Entity через `z.infer`.

## Почему

Ручной type рядом с zod-schema = два источника правды. Они расходятся **молча**: schema поменялась, type — нет; TS не подсказывает, потому что оба валидны сами по себе. `z.infer` физически не даёт разойтись — тип пересчитывается от той же схемы, которой валидируем рантайм. Это тот же принцип, что [modules-no-crutches](modules-no-crutches.md) (совместимость — свойство дизайна, не дисциплины памяти), применённый к типам: consistency гарантируется структурно, а не «не забудь обновить оба места».

## Где именно это применяется

- **Entity** — схема `zod` реальная (не plain config), single source of truth сущности ([anatomy](../packages/anatomy.md)).
- **View/Shape** — ноль явных импортов, ноль ручных типов ([import-rules](../architecture/import-rules.md)).
- **Contracts** (`commons/contracts`) — граница между репо тоже typed через zod, не через ручной `interface` шва ([modules-no-crutches](modules-no-crutches.md)).

## Стоп-сигнал

Соблазн написать `interface Props { ... }` «просто для этого компонента» → это симптом, что сущность/проп не выведен из своей zod-Entity. Останавливаемся, находим/заводим схему, берём тип оттуда — не пишем руками.
