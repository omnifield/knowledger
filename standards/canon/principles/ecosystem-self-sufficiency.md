# Ecosystem self-sufficiency — сторонние API = уровень интеграций

Верхний закон — `../../POLICY.md`. Родственные: [modules-no-crutches](modules-no-crutches.md)
(независимость модулей — тот же закон на уровне пакетов), [dogfood-product-flow](dogfood-product-flow.md)
(наш флоу = флоу юзера). Арх-контекст — ADR 078 §4 оракула (capability/provider/router).

## Тезис

**Все наши продукты самодостаточны и зависят только от нашей экосистемы. Любой сторонний
API/SDK/сервис — уровень интеграций.** Ядро продукта провайдер-агностично; сторонняя
возможность подключается адаптером, который мапит чужое в **наш** контракт — никогда
наоборот.

## Правила

1. **Контракт — свой (ecosystem-native).** Словарь сущностей/событий/операций проектируем
   мы, из потребностей ядра — не из shape чужого API. Чужой API реализует наш контракт
   через адаптер.
2. **Граница адаптера непрозрачна.** Типы, события, семантика и понятия вендора
   (SDK-типы, issue-номера, поля чужих борд) НЕ пересекают границу адаптера — не текут в
   kernel/ядро/UI. Использовать вендорский SDK *внутри* адаптера — ок; протечка наружу —
   нарушение.
3. **Тест на чистоту:** добавление/смена интеграции не трогает ничего, кроме регистрации
   нового адаптера. Если пришлось править ядро — контракт был сформован вендором, чиним
   в корне.
4. **Прагматика не отменяется.** MVP может гонять одну-единственную интеграцию (и не иметь
   ещё «своей» реализации) — но ядро об этом не знает: контракт с первого дня свой,
   интеграция = первый адаптер.

## Прецеденты (2026-07-08)

- **brainer** — ядро ориентировано на self-hosted agent-loop (наша эко); Claude API/SDK =
  интеграция-провайдер. Контракт control-канала — свой, в kernel; claude-agent-sdk живёт
  только внутри claude-code-адаптера.
- **tasker** — контракт task-native; GitHub Projects = одна из интеграций (v0-адаптер).
- **chater** — generic-мессенджер, про агентов вообще не знает; агент = participant через
  мост на стороне brainer.

## Why

Продукт, чей контракт сформован чужим API — заложник вендора: смена условий/цен/deprecation
чужого API = переписывание ядра. Self-host-first и федеративная ресурсно-тарифная модель
(ADR 072/078) требуют, чтобы возможность исполнялась на любом провайдере — это возможно
только при провайдер-агностик ядре. Юзер может хотеть жить вообще без сторонних сервисов
(и без нейронок) — продукт обязан оставаться целым.
