# UiProxy и тег-флоу событий

Механика, которой держится «UI is a Shadow»: UI не вызывает логику напрямую — события всплывают к логике по тегам через проксированный `Ui`.

## UiProxy

Когда `View` или `Shape` рендерится внутри `Controller`, базовый UI-kit оборачивается в Proxy. Политика **C — own meta opt-in**: побочные эффекты (регистрация в store, event-binding) активируются **только** если на JSX-узле явно задан `meta`. Структурные обёртки (`Field`, `Field.Label` и т.д.) проходят сквозным рендером.

Для узла с `meta`:
- стабильный `id` (`createUniqueId`) + `createEffect` для re-register на изменение props + `onCleanup` для unregister (нет утечки регистрации);
- автоподписка на 6 событий: `onClick`, `onInput`, `onChange`, `onBlur`, `onFocus`, `onKeyDown`;
- дедупликация bubbling через event-marker (один клик → один вызов handler);
- инжект реактивных `class` (+`store.styles[name]`), `disabled` (из `store.loading`), `name` (из `meta.tags`).

## Тег-флоу (децентрализованный N→1)

Инпуты несут **базовые семантические теги**; событие всплывает к хендлеру-владельцу **по тегу** (как нативное всплытие). Хендлер ищется: `schema.states[current][method]` → `schema[method]` (top-level) → `await next()` (автобабблинг к родителю).

- **Пакетные блоки** работают так же: пакет-View с тегом ловится тег-хендлером app-Feature. Пакет теги **не навязывает** — апп может перемапить (aliases).
- **Alias — query-side**: `@input` живёт в `pick`/`values`, НЕ в `meta.tags`.
- Поиск N→1 децентрализован — много источников, один обработчик, без явной проводки each-to-each.

## Именованные события (opt-in)

`useEmit`/типизированные события — **только** для семантики, которую надо типизировать (`onLogin`, `onImport`, `onSegmentNavigate`), НЕ на каждый клик. Оба канала живут одновременно: тег-флоу для «нативной» механики, именованные — для доменных швов между зонами.

## Правило

Не проводи UI→логику руками. Дал узлу `meta` с тегом — событие само найдёт владельца. Нужна типизированная доменная семантика — заведи именованное событие, не расширяй тег-флоу спец-случаями.
