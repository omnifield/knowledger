# Docs hygiene

Лечит v1: дубли, противоречия, «актуальность по датам», важное уезжает в угол и всплывает после граблей (ADR 077 §2).

## Правила

1. **Один источник на тему.** Появился дубль → слить в один, остальное пометить `superseded`. Два конкурирующих документа на одну тему = баг.
2. **Явный `superseded`.** Устаревшее не удаляем молча и не оставляем конкурировать — помечаем во frontmatter `status: superseded` + `superseded_by: <path>`. Читатель сразу видит, куда идти.
3. **Без дата-археологии.** «Новее дата = правда» — запрещённый способ разрешения конфликта. Правда = помеченный `current`-источник, не самый свежий файл.
4. **AI-anchor + user-guide split.** На фичу: AI-anchor (`docs/_meta/<x>.md` — углублённая арх для агента) + user-guide (для человека) где применимо. Не смешивать аудитории в одном файле.
5. **Линк, не копируй.** Cross-ref — ссылкой на источник, не дублированием текста (дубль расходится).
6. **Термины — в глоссарий** (`../../docs/glossary.md`), не переопределяй локально.
7. **Чистый markdown, ноль приложение-специфичных расширений.** Никаких Obsidian-wikilinks (`[[...]]`), эмбедов (`![[...]]`), тегов и прочего non-standard синтаксиса. Cross-ref — только обычная markdown-ссылка `[текст](относительный/путь.md)` — читается любым рендерером (GitHub, plain-text редактор, свой докс-движок) без привязки к конкретному приложению.
8. **Докс-движок не имеет права менять поведение доки.** Свой движок доков (заточенный под markdown) может добавлять стили/навигацию/поиск сверху — это ок. Но он не имеет права требовать нестандартный синтаксис для базовой работы (ссылки/структура) так, что дока ломается при открытии в другом окружении (GitHub, редактор, plain-text). Стилизация — да, зависимость поведения от конкретного движка — нет.

## Frontmatter-минимум

```yaml
---
title:
status: current | superseded | draft
superseded_by:      # путь, если status: superseded
owner-repo:         # чей это документ
---
```

## Где что живёт

- **Канон/дисциплина** (cross-ecosystem) → `commons/standards`.
- **Глоссарий + cross-cutting ADR** → `commons/docs`.
- **Per-repo арх/доки** (AI-anchor конкретного пакета, per-repo ADR) → **в своём репо**, не в commons.
