# Definition of Ready / Done

Ядро дисциплины (ADR 077 §2). Лечит kill-case v1: «уроки залили → решили менять → валидатор пишем на ходу → БД/фронт не готовы → контент льётся мимо валидаторов». Причина — не было **DoR по стадиям**. Здесь — гейты, без которых item не начинают и не закрывают.

## Пайплайн фичи (стадии, cross-repo)

Порядок обязателен: **downstream не стартует, пока upstream не Done.** Не каждая фича проходит все стадии; но если проходит — только в этом порядке.

```
Contract → Backend → Domain(framework) → App → DevOps
```

| Стадия | Репо | DoD стадии = «апстрим готов» для следующей |
|---|---|---|
| **Contract** | commons/contracts | схема+типы опубликованы `@omnifield/contract-*`, версия зафиксирована |
| **Backend** | backend | эндпоинт реализует контракт, тесты против контракта зелёные, сервис поднят |
| **Domain** | framework | пакет потребляет контракт (мок бэка = контракт), testbed-механика зелёная |
| **App** | apps | продукт композирует домен, prod-условия e2e зелёные |
| **DevOps** | devops | роут/deploy, gateway отдаёт, self-host ok |

Пайплайн — почему нужен контракт-first: домен и бэк могут идти **параллельно** от общего контракта (домен мокает бэк контрактом), вместо «бэк не готов → фронт ждёт / пишет наугад».

## DoR — можно НАЧИНАТЬ item

- [ ] **Upstream Done** — контракт есть, если я его потребляю; бэк живой, если я апп.
- [ ] **Дизайн/ADR согласован** — нет открытой арх-развилки.
- [ ] **Owner + scope ясны** — какой репо, какая зона.
- [ ] **Критерии приёмки записаны** — что конкретно значит «работает».
- [ ] **Нет блокирующих unknowns** — «не знаю работает ли» → сначала проверить (source/тест), не кодить.

Не выполнено → item в roadmap = `blocked`, не `doing`.

## DoD — можно ЗАКРЫВАТЬ item (= эталон-гейт)

- [ ] **Код** смержен в свой репо.
- [ ] **Тесты** — unit + (framework) testbed-механика / (app) prod-условия e2e. **Закрытие = автотесты зелёные + проверка в браузере** (product owner eyeball) — это и есть e2e-пруф прогона в реальности, не только «тесты зелёные».
- [ ] **Трейсы** — фича **инструментирована перф-логгерами** (`@omnifield/*` profiler): критические пути пишут observability на въезде, не «допишем позже». Столб гейта отдельный от тестов — наблюдаемость first-class.
- [ ] **Контракт** (если шов) опубликован и версионирован.
- [ ] **Доки** обновлены (AI-anchor + user-guide где применимо), старое помечено `superseded`.
- [ ] **Downstream разблокирован** — контракт опубликован / owner уведомлён.

## Зрелость item (значения в roadmap)

```
proposed → ready (DoR met) → doing → done (DoD met) → etalon
                                                     ↘ parked / superseded (оракул)
```

- `ready` ≠ `doing`: DoR выполнен, но ещё не в работе.
- `done` ≠ `etalon`: DoD выполнен для стадии; `etalon` — фича целиком доведена по всем своим стадиям.
- `parked` — сознательно отложено (доводим в оракуле / позже); не теряем из виду, но не блокирует.
