# Roadmap — живой борд (доктрина)

Один источник правды «где всё сейчас» — **GitHub Project** (org-level, поверх всех репо),
не markdown. **Никакой дата-археологии** — статус читается из полей карточки.

## Борд

🔗 https://github.com/orgs/omnifield/projects/1

Один org-борд на всю экосистему (не по борду на репо — иначе cross-repo цепочка теряется).
Карточки — issue из репо-владельцев, поля отражают стадию.

**Поля:** `Stage` (Proposed/Ready/Doing/Blocked/Done) · `Layer` (Contract/DevOps/Backend/App/Frontend)
· `Repo` (авто) · `Queue` (WIP=1) · `Blocked-by` · `DoR` / `DoD`.

## Как вести

- Одна карточка = один item (фича / дыро-фикс / контракт-решение).
- Двигай `Stage` / `Blocked-by` при каждом сдвиге — в паузах, перед рестартом сессии.
- `Blocked-by` пуст → можно двигать. Заполнен → ждёт апстрим (`definition-of-ready.md`).
- Закрыл item → `Stage: Done` + закрой issue (`-r completed`), не удаляй (история в issue).
- `Queue` — только для активной очереди запусков; завершённые milestone без номера.

## Карточка = issue (двухуровневый бриф)

Тело issue:

```
## 🧭 TL;DR (для тебя)
- 2–3 тейка: что за проблема, куда фикс, вывод. БЕЗ деталей.

---

## 🔧 Brief (исполнителю)
Факты / Fix (направление) / Verify (DoD) / Вне скоупа / Заметки (git).
```

- User оценивает по TL;DR, исполнитель работает по бриф-секции.
- Вопросы и обсуждение — **комментами** в issue, не в переписке (история сохраняется).

## Правило чтения (WIP=1, foundation-first)

Бери верхний item своей роли с пустым `Blocked-by` и минимальным `Queue`.
Очередь строго последовательная: следующий стартует, когда предыдущий закрыт.
Дыры-фиксы гейтят всё выше по иерархии базы.
